GitHub Cheat Sheet

Below is a handy cheet sheet I created to use whenever I need to commit and push local changes to GitHub.

Step 1: Open a terminal window.

Step 2: Change directory into your Git repository:

   ~ :> cd Documents/DBC/code/phase-0/

Step 3: Pull down any new commits (along with the new, deleted, and modified files) from GitHub to your computer:

   phase-0 [master] :> git pull

Step 4: Check out a new feature branch to work on:

   phase-0 [master] :> git checkout -b branch_name

Step 5: Do some work, son!

Step 6: Add any new or modified files to the index so they can be committed:

   phase-0 [branch_name] :> git add .

Step 7: Store the current contents of the index in a new commit with a log message:

   phase-0 [branch_name] :> git commit -m "Log message here"

Repeat steps 5-7 until you're done working.

Once you're done working:

Step 8: Send all the commits (along with the new, deleted, and modified files) to GitHub:

   phase-0 [branch_name] :> git push origin branch_name

Step 9: Create a pull request on GitHub.

Step 10: Review the changes and if satisfied, merge the feature branch into the master branch.

Step 11: Delete the feature branch on GitHub.

Step 12: Check out your local master branch:

   phase-0 [branch_name] :> git checkout master

Step 13: Pull down any new commits (along with the new, deleted, and modified files) from GitHub to your computer:

   phase-0 [master] :> git pull

Step 14: Delete your local feature branch:

   phase-0 [master] :> git branch -d branch_name

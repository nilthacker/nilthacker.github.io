Arrays and Hashes in Ruby

And, when to use each

25 October 2015

Arrays

An array is an ordered list of elements of any type. An array is defined by placing a list of elements within brackets:

  new_array = [1, "two", 3.0, "four"]

The array above contains an integer, a float, and some strings. You can place anything in an array, including another array.

An array is what's known as an indexed list, which means that every slot in an array is numbered and can be referenced by its index. It's important to note that the first array element is at index 0. So if we wanted to get to the second element in an array, we would access index 1. To access a specific element in array, we do this:

  new_array[2]

Which returns the third element in the array, 3.0.

Arrays are a very useful way of storing data that needs to be kept in order. When an element is added to an array, it is placed at the end of the list. Ruby's array class has a lot of very useful built-in functions that can be used to perform many of the actions that programmers use.

Hashes

A hash is a data structure that stores elements by their associated keys, unlike an array which stores items by an ordered index. Elements in a hash are frequently referred to as key-value pairs.

Hashes are most often created using symbols as keys with any data type as a value. All key-value pairs in a hash are surrounded by curly braces { } and separated by a comma.

Hashes are created by:

agent_bond = {
  firstname: "Bond"
  lastname: "James Bond"
  codename: "007"
}
miss_moneypenny = {
  firstname: "Eve"
  lastname: "Moneypenny"
  codename: "Moneypenny"
}

To access an element stored an a hash, we simply put the key in between square brackets [ ], just like an array. If we need to retreive Bond, James Bond's codename:

  agent_bond[:codename]

Returns the value associated with the :codename key, 007.

When to use an array or a hash

Arrays are best used when things need to be stored in any sort of order, whereas hashes are useful when things need to be labeled. Things to keep in mind when deciding to use an array or a hash are:

  • If order matters, use an array. Such as eeping a queue, or mimicking a "first in first out" or "last in first out" stack.
  • If the data needs to be associated with a specific label, use a hash.
